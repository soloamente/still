## Learned User Preferences

- Short replies **go** / **ok** are used to advance Executor work and to mark Planner or human verification on Track B milestones in the scratchpad.
- Product direction favors a **Letterboxd-class diary and social layer** with a **Mobbin-informed web design system** (usable controls on top of cinematic chrome), not replacing the theater identity wholesale. When researching UI patterns via Mobbin, prefer **`image_format: "jpg"`** for `search_screens` because WebP responses can fail to decode in some agent or tooling environments.
- On **`/home`**, keep **sticky browse + search** (`HomeStickyChrome`), **remember catalogue chips** (sort, venue, movies vs TV, TV **`run`**) across navigation, and **prompt for watch region** on first use (**All countries** optional); **Movies**: left rail **Upcoming · Latest · Popular**, venue on the right when **Upcoming**; **TV**: left **Latest · Popular** only, right **Ongoing · Completed · Upcoming** as mutually exclusive **`?run=`** slices (not combined with each other; legacy **`?sort=ongoing|upcoming`** maps to **`run`**); **In cinemas / At home** on TV only when **`run=upcoming`**; **Continue watching** is the signed-in **`tv_watch`** rail on **`browse=tv` only** (not Movies), distinct from public **Ongoing** (TMDb **Returning Series**, not `on_the_air` — avoids overlap with **Completed**); **movie detail** browse-back restores the last catalogue context (movies vs TV label and filters), not always default **Movies** / **`/home`**.
- **Theaters** means **in-cinema** availability and **Streaming** means **watch at home**; venue filters and TMDB-backed lists should preserve that distinction with strict **Upcoming** vs **Latest** separation (including no unreleased titles in **Latest**).
- When **`browse=community`**, secondary controls are **Lists**, **Reviews**, and **Activity** only (no separate **Diary** chip — watch logs live in **Activity**); use **full-height centered empty states** when a tab has no rows.
- **Overlays, chrome, movie/TV detail, and search dialogs** should lean on **surface depth tokens**, not **borders, rings, shadows, or decorative outlines**; **movie detail** pages use a **`bg-card`** inner wrapper like catalogue lobbies. **Community feed rows** (`ActivityItem`, `ReviewCard`, friend rail) use flat **`bg-background`** tiles on **`bg-card`** — no row borders and **no box-shadow**; **`ActivityItem`** matches review width (**`max-w-2xl`**) — poster (**`FeedListingThumb`** **`layout="activity"`**: **`w-16`** **`aspect-2/3`** **`self-start`**) | byline (**`FeedPersonAvatar`** + name + inline **`FeedActivityVerb`** + time) | title | meta — **no side icon** chrome.
- **Patron ratings** display on a **0.0–10.0** scale (`log.rating` / `review.rating` stored as integer **tenths** 0–100, legacy 1–10 as whole scores). Use **`logRatingToDisplay`**, **`DiaryLogRatingLabel`**, or **`formatStoredLogRatingDisplay`** — never pass raw stored values to **`StarRating`** or **`formatLogRatingDisplay`** alone (shows bogus values like **47.5**). **`log.liked`** on watch logs means **favorites** (heart chip label **Favorite**, **`IconHeartFilled`**), not a generic “liked” reaction.
- **Patron portraits** in feed / rail / nav: **`PatronPortraitAvatar`** + **`profilePatronAvatarImageUrl(handle)`** (proxies private Vercel Blob) — not raw **`user.image`** in Next **`<Image>`**.
- **Avatar account menu**: menu item text should be **16px** with **`px-5`** inset rows; primary routes (**Achievements**, **Log out**) use **semibold** weight — no **Request feature**, **Give feedback**, or in-menu **Theme** segment (theme stays in **Settings**). Row hover/active backgrounds should **inset within the dropdown padding** (not full-bleed to the popover edge).
- For **diary logging and edits**, capture **watch venue** (cinema vs at home); **default new logs to at home**. **TV** uses hybrid **`tv_watch`** progress (statuses **watching · paused · abandoned · finished · rewatching**) plus **scoped diary** on `log` (**`show` / `season` / `episode`**); on **`/tv/[id]`**, **Watching** status and **By season / By episode** use **`SegmentedPillToolbar`** (same **`rounded-full bg-background`** track + sliding **`bg-card`** pill as **`/home`** chips, centered); **Quick Log** scope/season/episode pickers use **`StillPopoverSelect`** and the same capped scroll + bottom gradient scrim as **`review-composer.tsx`** (`max-h-[min(calc(92svh-11rem),640px)]`, `pb-24`, footer `z-20`).
- **Movie detail hero**: titles use **SF Pro Rounded** (**`font-sans`**), not Fraunces **`font-display`**; before first log, **watchlist** (left) + **Add to Watched** pill; after logging, watchlist exits, pill morphs to **rewatch** circle, **Add review** is the primary pill, row stays **centered** (no **More actions** menu).
- **Awards & festivals** on movie detail (not “Premieres”): **Mubi-style** centered section with a **horizontal brand-logo row** (up to **12** entries in a **6-column**, two-row grid on large screens); each column title is the **festival/award name**; detail lines show **year on one line** and **winner/nominee achievement on the next** (no `year | detail` pipe); use **`text-balance`** on titles and award copy; leave **generous vertical gap** after **Cast & Crew**; **cast & crew arc** (`movie-cast-crew-arc`): **odd** visible slot count (single center apex), **normalized parabola** curve, row **padding** matching arc offset so outer cards don’t overlap the opposite row; keep cast/crew rows **tight** (content-sized — avoid tall `min-h` + `justify-center` spreading rows); hero **synopsis blurb** scales wider on large viewports (beyond **`max-w-prose`**).
- **Sticky search** (`HomeStickySearch`): **inline scrollable poster results** in the anchored dialog (no standalone **`/search`** page); global shortcut opens it on **any app page** with the same expand animation as clicking the bar on **`/home`**; search field has **no focus outline** and avoids **orange/accent** chrome; **`SearchTokenField`** + **Tab** pills (**studio** logo, **Films/TV**, **genre**, curated e.g. **Anime**, **lists** = signed-in user's lists only) via **`search-query-tags.ts`**, then free-text AND filters; **Films / TV** toggle hides when a media pill is set; empty-state **studio** chips use **`bg-*` tiles** for logo contrast — **no border/ring** on chips; empty state uses left-rail categories with a **4-poster** preview column. **Movie/TV detail**: **Share** copies URL to clipboard only; press motion on **back**, **share**, hero actions, and **save** — **not** About/Streaming **tabs**; **`bg-background`** controls on **`bg-card`** use **`DETAIL_CANVAS_ON_CARD_HOVER_CLASS`** (`detail-action-motion.ts`), not **`hover:bg-muted/*`**; **Streaming** tab uses **Mobbin-style** scrollable provider rail, **checkmarks** (not price pills), **`nucleo-flags`** on country rows. **`AppScrollToTop`**: hover **width-expand** with a **fixed trailing icon slot** and clear label–icon gap — animate **width** only, not gap/padding/scale together.
- **`/lists`**: personal-only poster-wall lobby like **`/home`** / diary / watchlist (**`HomeStickyChrome`** includes a **lists** shortcut); lobby posters use shared **`RadialToolkit`** (**`useRadialToolkitAnchor`**: hold **RMB**, drag to aim, release to confirm; suppress native **context menu** after release) with **Nucleo** icons — **destructive** wedge/label styling **only** on **Delete** (no **Share** slot); lobby actions: **Edit** (title + description), **Change cover** (device file picker → **`uploadListCover`**), **Delete** (confirm dialog aligned with account-leave pattern); list modals use **`APP_MODAL_OVERLAY_CLASS`** **`z-[250]`** above toolkit **`z-[200]`**; **list detail** mirrors **movie detail** (`bg-card`, section nav, centered hero); owners set **custom cover** (pinned title or upload); lobby tiles show **title bottom-center** on the poster scrim.

## Learned Workspace Facts

- The repo is a **monorepo**: main web app under **`apps/web`** (Next.js App Router), API under **`apps/server`**, shared UI and global styles under **`packages/ui`** (for example `packages/ui/src/styles/globals.css`).
- **Track B** (design system and screen IA) is the canonical plan for navigation, tokens, search and browse primitives, and core screens; it lives in **`.cursor/scratchpad.md`** as tasks **B.1–B.7** alongside Phase 8 polish items.
- **Elevation tokens** use an explicit ladder: **`surface-canvas`** → **`surface-raised`** → **`surface-overlay`**, mapped through shadcn-style **`--background`**, **`--card`**, and **`--popover`** for predictable depth on dark UI and **`.movie-themed`** pages.
- The authenticated app chrome is centralized in **`apps/web/src/components/app/app-shell.tsx`** (`AppShell`): **`main#main-content`**, shared horizontal gutters, and exports **`APP_SHELL_BOTTOM_RESERVE_CSS`** / **`appShellMainContentMinHeightStyle`** for pages that must match the nav inset (for example **`people/[id]`**). Bottom padding for `#main-content` is **`10px`** in **`packages/ui/src/styles/globals.css`** (same literal as **`APP_SHELL_BOTTOM_RESERVE_CSS`**).
- **Track B.4** introduced shared primitives such as **`SearchPillField`**, **`FilterChipRow`** / **`FilterChipLink`** / **`FilterChipButton`**, **`MovieCatalogSurfaceChips`**, and catalogue routes **`/movies/popular`**, **`/movies/upcoming`**, and **`/movies/discover`**; chip hrefs and discover fetches stay aligned via **`apps/web/src/lib/discover-catalog-url.ts`** (canonical sort + genre query building).
- **`/home`** (Track B.5.1): **`HomeStickyChrome`** pins browse/search; **`HomeStickySearch`** opens an anchored dialog (inline poster grid via **`use-catalog-text-search`** / **`use-catalogue-tag-search`**, **`SearchTokenField`**, **`search-query-tags.ts`**, **`/search` retired**); **Community** tab **`HomeCommunityLobby`** (**`home-community-feed.ts`**, **`GET /api/feed`**, **`/api/logs/recent`**, **`/api/reviews/recent`**) with **`ActivityItem`** + **`FeedPersonAvatar`**; friend-activity rail from **`lg`** (**`HomeFriendActivityRail`**, **`home-friend-rail.ts`**). TV lobby: **`home-catalog-run.ts`** (`?run=ongoing|completed|upcoming`), **`parseTvLobbyVenue`** in **`home-venue.ts`**, chips in **`home-catalog-sort-chips.tsx`** + **`home-catalog-view-mode-toolbar.tsx`**; **`GET /api/tv/on-the-air`** and **`/tv/on-the-air`** for the full **Ongoing** catalogue; feed thumbs via **`tmdb-poster-url.ts`**.
- Import UI motion from **`motion/react`**, not **`framer-motion`** (package is **`motion`**); shared detail press springs live in **`apps/web/src/lib/detail-action-motion.ts`** and **`detail-motion-pressable.tsx`**.
- **`/tv/[id]`** reuses movie detail shell components; server **`GET /api/tv/:id`**, seasons routes, **`tv-watch`** CRUD (`apps/server/src/routes/tv-watch.ts`, **`tv-season-cache.ts`**), **`ensureTvCached`**, and logs/watchlist rows with **`tvId`** (exactly one of **`movieId`** / **`tvId`**); web **`tv-detail-primary-actions.tsx`**, **`tv-log-scope-picker.tsx`**, **`StillPopoverSelect`**. **Lists**: **`lists-lobby-catalogue.tsx`**, **`list-lobby-poster.tsx`**, **`lists-lobby-order.ts`**, **`list-detail-*`**, **`list-display-cover.ts`**, **`list-cover-image.ts`**, **`upload-list-cover.ts`**, **`vercel-blob-image-put.ts`**; shared **`RadialToolkit`** in **`packages/ui/src/components/radial-toolkit.tsx`**; modal stacking via **`apps/web/src/lib/app-modal-layer.ts`** (**`APP_MODAL_OVERLAY_CLASS`**).
- If **`bun run build`** in **`apps/web`** fails TypeScript on **`Link`** **`href`** values or route types (**`RouteImpl`**) even though **`typedRoutes`** is false, delete **`apps/web/.next`** and rebuild; stale generated types can disagree with the current **`app/`** tree.
- Profile filmography timestamps may arrive as a **`Date`** or an **ISO string**; UI that formats years or dates should **normalize** unknown timestamp shapes instead of assuming **`.slice`** on a string. For catalogue infinite grids (e.g. **`popular-movies-infinite`**), when a reset **`useEffect`** must track a **composite catalogue surface** but Biome **`useExhaustiveDependencies`** discourages listing overlapping primitives, derive a single string key (e.g. **`catalogueWaveKey`**) and reference it inside the effect (e.g. **`void catalogueWaveKey`**) so dependencies stay explicit without splitting unrelated concerns.
- **`/diary`** and **`/watchlist`** mirror **`/home`** **catalogue lobby shell** (**`HomeStickyChrome`**, poster grid); **diary venue chips filter the diary route in place** rather than redirecting to **`/home`**. **Legacy** log rows missing **`watch_venue`** stay discoverable by **coercing** a display value and **surfacing them in both** venue slices until the user edits the entry.
- **Festival brand icons** in **`packages/ui/src/icons/`** (e.g. **`mtv.tsx`** from **`mtv.svg`**); recognition rows from **`movie-festival-recognition.ts`** + **`wikidata-movie-awards.ts`**, rendered in **`movie-premieres-festivals.tsx`**. **Cast & crew arc** layout math in **`movie-cast-crew-arc.ts`** (`CAST_CREW_ARC_EDGE_OFFSET_PX`, `arcVisibleSlotCount`, `reorderForCenterArc`). **Streaming** tab: **`movie-detail-streaming.tsx`**, **`movie-watch-providers.ts`**, **`nucleo-flag-by-iso.ts`**. **Movie detail About** uses fixed **`MovieDetailSectionNav`** (right rail) and **`MovieDetailBodySection`** with symmetric **`xl:px-28`** gutters for the legend.
