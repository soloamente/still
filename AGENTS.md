## Learned User Preferences

- Short replies **go** / **ok** are used to advance Executor work and to mark Planner or human verification on Track B milestones in the scratchpad.
- Product direction favors a **Letterboxd-class diary and social layer** with a **Mobbin-informed web design system** (usable controls on top of cinematic chrome), not replacing the theater identity wholesale. When researching UI patterns via Mobbin, prefer **`image_format: "jpg"`** for `search_screens` because WebP responses can fail to decode in some agent or tooling environments.
- On **`/home`**, keep **sticky browse + search** (`HomeStickyChrome`), **remember catalogue chip selection** (sort, venue, movies vs TV) across navigation, show **Upcoming before Latest** for movies and TV, and **prompt for watch region** on first use (**All countries** optional); **movie detail** browse-back restores the last catalogue context (movies vs TV label and filters), not always default **Movies** / **`/home`**.
- **Theaters** means **in-cinema** availability and **Streaming** means **watch at home**; venue filters and TMDB-backed lists should preserve that distinction with strict **Upcoming** vs **Latest** separation (including no unreleased titles in **Latest**).
- When **`browse=community`**, secondary controls should emphasize **community surfaces** (e.g. **Lists**, **Reviews**, **Activity**) instead of catalogue **Latest/Popular** chips; use **full-height centered empty states** when a tab has no rows.
- **Overlays, chrome, movie/TV detail, and search dialogs** should lean on **surface depth and shadow tokens**, not **borders, rings, or decorative outlines**; **movie detail** pages use a **`bg-card`** inner wrapper like catalogue lobbies.
- **Avatar account menu**: menu item text should be **16px**; row hover/active backgrounds should **inset within the dropdown padding** (not full-bleed to the popover edge).
- For **diary logging and edits**, capture **watch venue** (cinema vs at home); **default new logs to at home**; **TV** diary, **watchlist**, and **quick-log** should mirror **films** (TMDb **`tv_id`** on log/watchlist rows).
- **Movie detail hero**: titles use **SF Pro Rounded** (**`font-sans`**), not Fraunces **`font-display`**; before first log, **watchlist** (left) + **Add to Watched** pill; after logging, watchlist exits, pill morphs to **rewatch** circle, **Add review** is the primary pill, row stays **centered** (no **More actions** menu).
- **Awards & festivals** on movie detail (not “Premieres”): **Mubi-style** centered section with a **horizontal brand-logo row** (up to **12** entries in a **6-column**, two-row grid on large screens); each column title is the **festival/award name**; detail lines show **year on one line** and **winner/nominee achievement on the next** (no `year | detail` pipe); use **`text-balance`** on titles and award copy; leave **generous vertical gap** after **Cast & Crew**; **cast & crew arc** (`movie-cast-crew-arc`): **odd** visible slot count (single center apex), **normalized parabola** curve, row **padding** matching arc offset so outer cards don’t overlap the opposite row; keep cast/crew rows **tight** (content-sized — avoid tall `min-h` + `justify-center` spreading rows); hero **synopsis blurb** scales wider on large viewports (beyond **`max-w-prose`**).
- **Sticky search** (`HomeStickySearch`): **inline scrollable poster results** in the anchored dialog (no standalone **`/search`** page); non-empty query supports **Films / TV** filter; empty state uses left-rail categories with a **4-poster** preview column. **Movie/TV detail**: **Share** copies URL to clipboard only; press motion on **back**, **share**, hero actions, and **save** — **not** About/Streaming **tabs**; **`bg-background`** controls on **`bg-card`** use **`DETAIL_CANVAS_ON_CARD_HOVER_CLASS`** (`detail-action-motion.ts`), not **`hover:bg-muted/*`**; **Streaming** tab uses **Mobbin-style** scrollable provider rail, **checkmarks** (not price pills), **`nucleo-flags`** on country rows. **`AppScrollToTop`**: hover **width-expand** with a **fixed trailing icon slot** and clear label–icon gap — animate **width** only, not gap/padding/scale together.

## Learned Workspace Facts

- The repo is a **monorepo**: main web app under **`apps/web`** (Next.js App Router), API under **`apps/server`**, shared UI and global styles under **`packages/ui`** (for example `packages/ui/src/styles/globals.css`).
- **Track B** (design system and screen IA) is the canonical plan for navigation, tokens, search and browse primitives, and core screens; it lives in **`.cursor/scratchpad.md`** as tasks **B.1–B.7** alongside Phase 8 polish items.
- **Elevation tokens** use an explicit ladder: **`surface-canvas`** → **`surface-raised`** → **`surface-overlay`**, mapped through shadcn-style **`--background`**, **`--card`**, and **`--popover`** for predictable depth on dark UI and **`.movie-themed`** pages.
- The authenticated app chrome is centralized in **`apps/web/src/components/app/app-shell.tsx`** (`AppShell`): **`main#main-content`**, shared horizontal gutters, and exports **`APP_SHELL_BOTTOM_RESERVE_CSS`** / **`appShellMainContentMinHeightStyle`** for pages that must match the nav inset (for example **`people/[id]`**). Bottom padding for `#main-content` is **`10px`** in **`packages/ui/src/styles/globals.css`** (same literal as **`APP_SHELL_BOTTOM_RESERVE_CSS`**).
- **Track B.4** introduced shared primitives such as **`SearchPillField`**, **`FilterChipRow`** / **`FilterChipLink`** / **`FilterChipButton`**, **`MovieCatalogSurfaceChips`**, and catalogue routes **`/movies/popular`**, **`/movies/upcoming`**, and **`/movies/discover`**; chip hrefs and discover fetches stay aligned via **`apps/web/src/lib/discover-catalog-url.ts`** (canonical sort + genre query building).
- **`/home`** (Track B.5.1): **`HomeStickyChrome`** pins browse/search; **`HomeStickySearch`** opens an anchored dialog (inline poster grid via **`use-catalog-text-search`**, **`/search` retired**); **`ActivityItem`** + **`FeedPersonAvatar`**; friend-activity rail from **`lg`** (**`HomeFriendActivityRail`**, **`home-friend-rail.ts`**).
- Import UI motion from **`motion/react`**, not **`framer-motion`** (package is **`motion`**); shared detail press springs live in **`apps/web/src/lib/detail-action-motion.ts`** and **`detail-motion-pressable.tsx`**.
- **`/tv/[id]`** reuses movie detail shell components; server **`GET /api/tv/:id`**, **`ensureTvCached`** (`apps/server/src/lib/tv-cache.ts`), and logs/watchlist rows with **`tvId`** (schema check: exactly one of **`movieId`** / **`tvId`**).
- If **`bun run build`** in **`apps/web`** fails TypeScript on **`Link`** **`href`** values or route types (**`RouteImpl`**) even though **`typedRoutes`** is false, delete **`apps/web/.next`** and rebuild; stale generated types can disagree with the current **`app/`** tree.
- Profile filmography timestamps may arrive as a **`Date`** or an **ISO string**; UI that formats years or dates should **normalize** unknown timestamp shapes instead of assuming **`.slice`** on a string. For catalogue infinite grids (e.g. **`popular-movies-infinite`**), when a reset **`useEffect`** must track a **composite catalogue surface** but Biome **`useExhaustiveDependencies`** discourages listing overlapping primitives, derive a single string key (e.g. **`catalogueWaveKey`**) and reference it inside the effect (e.g. **`void catalogueWaveKey`**) so dependencies stay explicit without splitting unrelated concerns.
- **`/diary`** and **`/watchlist`** mirror **`/home`** **catalogue lobby shell** (**`HomeStickyChrome`**, poster grid); **diary venue chips filter the diary route in place** rather than redirecting to **`/home`**. **Legacy** log rows missing **`watch_venue`** stay discoverable by **coercing** a display value and **surfacing them in both** venue slices until the user edits the entry.
- **Festival brand icons** in **`packages/ui/src/icons/`** (e.g. **`mtv.tsx`** from **`mtv.svg`**); recognition rows from **`movie-festival-recognition.ts`** + **`wikidata-movie-awards.ts`**, rendered in **`movie-premieres-festivals.tsx`**. **Cast & crew arc** layout math in **`movie-cast-crew-arc.ts`** (`CAST_CREW_ARC_EDGE_OFFSET_PX`, `arcVisibleSlotCount`, `reorderForCenterArc`). **Streaming** tab: **`movie-detail-streaming.tsx`**, **`movie-watch-providers.ts`**, **`nucleo-flag-by-iso.ts`**. **Movie detail About** uses fixed **`MovieDetailSectionNav`** (right rail) and **`MovieDetailBodySection`** with symmetric **`xl:px-28`** gutters for the legend.
